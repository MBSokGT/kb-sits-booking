# КБ Ситс — Система бронирования мест в коворкинге

❗️Собрано с помощью ИИ, я в этом энтузиаст любитель - так что любая критика, правки и дополнения только приветствуются❗️

Веб-система для бронирования рабочих мест в офисе с визуальным редактором планировки.

## 🌐 Продакшн

| Сервис | URL |
|--------|-----|
| Приложение | https://kb-sits.pages.dev |
| Репозиторий | https://github.com/MBSokGT/kb-sits-booking |

## ✨ Возможности

### 🔐 Авторизация
- Вход по email + пароль
- Саморегистрация отключена (аккаунты создаёт администратор)
- Смена пароля в личном кабинете
- Письма через Resend: приглашение при создании аккаунта, код сброса пароля, уведомление о смене пароля
- Автоматический выход после 60 минут бездействия
- Сессия восстанавливается после перезагрузки страницы

### 🗺️ Карта и планировка
- Загрузка планов этажей (JPG / PNG / PDF)
- Визуальное рисование зон мышью
- Настройка названия, количества мест и цвета

### 📅 Расширенный календарь
- Выбор диапазона дат (рабочие дни)
- Мульти-бронирование на несколько дней
- Временные слоты: утро / день / вечер / весь день / своё время
- Учёт рабочих суббот

### ⏰ Автоистечение броней
- Брони автоматически удаляются после окончания времени

### 👥 Роли и права

| Роль | Возможности |
|------|-------------|
| Сотрудник | Бронирование 1 места в день или на месяц вперёд |
| Руководитель | Бронирование за себя и за сотрудников отдела, просмотр бронирований отдела |
| Администратор | Все права выше + управление пользователями, планировками, департаментами, экспорт CSV |

### 🏢 Управление департаментами (Администратор)
- Список департаментов с сотрудниками
- Добавление / удаление / переименование отделов
- Назначение руководителей
- Добавление и исключение сотрудников

## 🛠️ Технологии

- **Frontend:** Vanilla JavaScript (ES6+), HTML5, CSS3, SVG
- **Backend:** Cloudflare Pages Functions (Edge Worker)
- **База данных:** Cloudflare D1 (SQLite)
- **Хостинг:** Cloudflare Pages
- **CI/CD:** GitHub → Cloudflare Pages (auto-deploy при push в main)

## 🔒 Доступ

Учётные записи управляются администраторами системы.
Для получения доступа обратитесь к администратору.

## 📦 Локальная разработка

```bash
git clone https://github.com/MBSokGT/kb-sits-booking.git
cd kb-sits-booking

# Локальный сервер (без D1 — localStorage-режим)
python3 -m http.server 8000

# Локальный сервер с D1 (требует wrangler + аутентификация)
npx wrangler pages dev . --d1 DB=e501ba2e-9bbf-4f1d-80c5-fad567368933
```

## ☁️ Деплой

```bash
# Задеплоить с D1 binding
npx wrangler pages deploy . --project-name kb-sits

# Применить изменения схемы D1
npx wrangler d1 execute kb-sits-db --file=d1-schema.sql --remote
```

## 🔐 Переменные окружения (Pages Functions)

- `CORS_ORIGINS` — список разрешённых origin через запятую  
  Пример: `https://kb-sits.pages.dev,https://your-domain.ru`
- `RESET_TOKEN_PEPPER` — секрет для хеширования кодов сброса пароля
- `RESET_DEBUG=1` — только для отладки, чтобы API возвращал код сброса в ответе
- `EMAIL_PROVIDER` — провайдер почты (сейчас поддерживается `resend`)
- `RESEND_API_KEY` — API ключ Resend
- `EMAIL_FROM` — подтверждённый отправитель в Resend, например `KB Sits <noreply@your-domain.ru>`
- `APP_BASE_URL` — публичный URL приложения для ссылки в письме (например `https://kb-sits.pages.dev`)
- `EMAIL_INCLUDE_PASSWORD=1` — опционально: включать пароль в письмо (по умолчанию `0`)

### Отправка письма при создании аккаунта админом

1. В Resend подтвердите домен отправителя и получите API key.
2. Добавьте секреты в Cloudflare Pages:

```bash
npx wrangler pages secret put EMAIL_PROVIDER --project-name kb-sits
npx wrangler pages secret put RESEND_API_KEY --project-name kb-sits
npx wrangler pages secret put EMAIL_FROM --project-name kb-sits
npx wrangler pages secret put APP_BASE_URL --project-name kb-sits
# опционально:
npx wrangler pages secret put EMAIL_INCLUDE_PASSWORD --project-name kb-sits
```

3. Задеплойте проект.
4. В админке при создании пользователя оставьте включённым чекбокс `Отправить письмо с доступом`.
