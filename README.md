# КБ — Бронирование рабочих мест

> ⚠️ Проект собран энтузиастом-любителем с помощью ИИ. Код не претендует на идеал — любая критика, правки и дополнения только приветствуются.

Веб-система для бронирования рабочих мест в офисе с визуальной картой планировки.

**Прод:** https://kb-sits.pages.dev
**Репозиторий:** https://github.com/MBSokGT/kb-sits-booking

---

## Функционал

### Авторизация
- Вход по email + пароль
- Саморегистрация отключена — аккаунты создаёт администратор
- Сброс пароля через email (код подтверждения)
- Смена пароля в личном кабинете
- Автоматический выход после 60 минут бездействия
- Сессия сохраняется после перезагрузки страницы

### Карта и планировка
- Интерактивная SVG-карта этажа
- Загрузка плана этажа (JPG / PNG / PDF)
- Визуальное рисование зон мышью прямо поверх плана
- Настройка названия зоны, количества мест и цвета
- Переключение между видом карты и списком

### Бронирование
- Выбор одной или нескольких дат (рабочие дни)
- Shift + клик для выбора диапазона дат
- Мультибронирование — одна зона на несколько дней сразу
- Временные слоты: утро (09–13), день (13–17), вечер (17–21), весь день (09–21), своё время (от 30 мин)
- Учёт рабочих суббот
- Проверка конфликтов: нельзя забронировать занятое место или пересекающийся слот
- Автоматическое истечение броней по времени (история хранится 1 год)

### Мои бронирования
- Список активных броней с датой, временем и зоной
- Выбор нескольких броней через чекбоксы и групповая отмена
- Отмена отдельной брони одной кнопкой

### Личный кабинет
- Просмотр своего профиля (имя, email, отдел, роль)
- Смена пароля
- Список своих активных броней

### Роли и права

| Роль | Что может |
|------|-----------|
| Сотрудник | Бронировать за себя, видеть свои брони |
| Руководитель | Бронировать за себя и за сотрудников своего отдела, видеть брони команды |
| Администратор | Всё + управление пользователями, планировками, коворкингами, отделами, экспорт |

### Вид «Команда» (Руководитель и Администратор)
- Список коллег из своего отдела
- Активные брони каждого сотрудника

### Административная панель

**Пользователи**
- Создание аккаунта (имя, email, отдел, роль, пароль)
- Опциональная отправка приглашения на email при создании
- Редактирование имени, email, отдела
- Смена роли
- Удаление пользователя
- Счётчики по ролям (сотрудники / руководители / администраторы)

**Планировки**
- Создание коворкингов и этажей
- Визуальный редактор зон: рисование, изменение размера, перетаскивание, удаление
- Загрузка плана этажа (JPG / PNG / PDF — PDF конвертируется автоматически)
- Удаление этажа или всего коворкинга вместе с привязанными бронями

**Бронирования**
- Полная статистика: всего броней, за неделю, за сегодня
- Список броней по дням с именем, зоной и временем
- Отмена любой брони
- Экспорт всех броней в CSV

**Отделы**
- Список отделов с участниками
- Создание, переименование, удаление отдела
- Назначение руководителя отдела
- Добавление и исключение сотрудников

## Технологии

- **Frontend:** Vanilla JavaScript (ES6+), HTML5, CSS3, SVG
- **Backend:** Cloudflare Pages Functions (Edge Workers)
- **База данных:** Cloudflare D1 (SQLite) — основная
- **База данных:** Supabase (PostgreSQL) — схема есть, на случай миграции
- **Хостинг:** Cloudflare Pages
- **CI/CD:** GitHub → Cloudflare Pages

## Переменные окружения (Cloudflare Pages)

| Переменная | Описание |
|-----------|----------|
| `CORS_ORIGINS` | Разрешённые origin через запятую |
| `RESET_TOKEN_PEPPER` | Секрет для хеширования кодов сброса пароля |
| `EMAIL_PROVIDER` | Провайдер почты (поддерживается `resend`) |
| `RESEND_API_KEY` | API ключ Resend |
| `EMAIL_FROM` | Отправитель, например `KB <noreply@domain.ru>` |
| `APP_BASE_URL` | Публичный URL приложения |
| `EMAIL_INCLUDE_PASSWORD` | `1` — включать пароль в письмо-приглашение |
| `RESET_DEBUG` | `1` — возвращать код сброса в ответе API (только для отладки) |

## Локальная разработка

```bash
git clone https://github.com/MBSokGT/kb-sits-booking.git
cd kb-sits-booking

# Без D1 — работает через localStorage
python3 -m http.server 8000

# С D1 (требует wrangler и авторизацию Cloudflare)
npx wrangler pages dev . --d1 DB=<database_id>
```
